## For the Sales Team

### Elevator Pitch (30 seconds)

> "The PushEngage Chrome Extension lets your customers create push campaigns from any webpage in 30 seconds. AI writes the copy, detects the page type, and generates smart CTAs automatically. It's a 6-10x speedup over the dashboard workflow."

---

### Key Selling Points

- **AI-powered copy generation** — 3 options with tone control, not just one generic suggestion
- **Smart CTA detection** — Recognizes 30+ page types: blog, product, video, podcast, recipe, event, e-commerce, and more
- **Content auto-extraction** — Pulls 15+ data points from any page (title, description, image, social links, and more)
- **Draft auto-save per URL** — Drafts persist across browser sessions scoped to the exact page
- **JTBD recommendations** — Guides users to their next best action based on account activity and goals

**Content Publishers**
- Every blog post becomes a one-click campaign with pre-filled title, excerpt, and CTA

**SaaS**
- Launch feature announcements directly from the feature landing page
- "Watch Demo" CTAs auto-detected on pages with embedded videos
- No need to switch context from content creation to campaign creation

**Media / News**
- Breaking news stories: publish to push in 30 seconds from the article page
- Journalists and editors can send campaigns without ever opening the dashboard
- Speed is the core value — the extension removes all friction

**Education / Courses**
- "Watch Course" and "Start Learning" CTAs auto-detected on course pages
- Promote new modules or live sessions directly from the course page

### Demo Script (2 minutes)

Follow this sequence for a clean, high-impact demo:

1. Open any blog page in Chrome
2. Click the extension icon — show the form auto-filling with the page's title, description, and image
3. Click "AI Write" — show all 3 copy options with distinct tones and smart CTAs
4. Point out social link detection appearing in the CTA button suggestions
5. Show the header nudge message (subscriber count, last campaign date)
6. Close the extension, reopen it — demonstrate that the draft was saved
7. Navigate to a product page — show the form refreshing with "Shop Now" CTAs

Key moments to highlight: the auto-fill on step 2 (saves time), the 3 AI options on step 3 (gives control), and the draft persistence on step 6 (reliability). End with the product page to show intelligence — the extension knows the page type changed.

## For Customer Success

### Onboarding New Users

Walk new users through this sequence on their first call or in the onboarding email:

1. Install the extension from the Chrome Web Store
2. Pin it to the Chrome toolbar (right-click the extension icon → Pin)
3. Click the pinned icon — it opens the PushEngage dashboard login in a new tab
4. Log in to the dashboard, then close that tab
5. Navigate to any webpage and click the extension icon — the form auto-fills
6. Click "AI Write" to generate copy options
7. Select an option and click Send — first campaign done

The goal of onboarding is one successful campaign before the call ends. Everything else is secondary.

---

### Common Support Scenarios

| Issue | Solution |
|-------|----------|
| "I can't log in" | The extension opens the dashboard login page. After logging in, close that tab and reopen the extension. The session carries over automatically. |
| "My draft disappeared" | Drafts are scoped per URL and expire after 24 hours. Ask the user to navigate back to the original page — the draft will reappear if it is within 24 hours. |
| "AI Write isn't working" | Check AI credits in the user's account settings. Free plans have a limited credit allocation. If credits are exhausted, prompt the upgrade path. |
| "I don't see Action Buttons" | Action Buttons require the Growth plan. The extension shows an inline upgrade prompt — point the user to that. |
| "I don't see Segments" | Segments require the Business plan. Same as above — use the inline prompt as the upsell entry point. |
| "Session expired" | This is expected behavior — 6-hour timeout for security. The user clicks the extension and is prompted to log in again. Takes under 30 seconds. |
| "CTA says Read More on a video page" | The extension detects 30+ page types including video and video courses. If it misidentifies a page, the user can manually edit the button text directly in the form. |
| "Extension shows blank" | Refresh the current page and reopen the extension. If it persists, check for browser extension conflicts or try disabling and re-enabling. |

### Health Indicators to Monitor

Use these signals to identify at-risk accounts and expansion opportunities:

| Metric | Healthy Range | Action if Outside Range |
|--------|--------------|------------------------|
| Subscriber count | Growing month-over-month | Investigate opt-in prompt placement |
| Campaign frequency | 8-12 campaigns per month | Below 8: re-engagement outreach; above 12: check for subscriber fatigue |
| CTR | Above 4% | Below 4%: review copy quality and send timing |
| Days since last campaign | 7 days or fewer | Over 7 days: triggers a re-engagement recommendation in the extension |

The extension surfaces some of these signals in the header nudge area. When a user sees "You haven't sent a campaign in 10 days," that is a CS-created touch point built into the product.

---

### Upsell Triggers in the Extension

The extension surfaces upgrade opportunities in three places:

**Insights page cards** — Shows "Unlock segments" and "Unlock A/B testing" cards when the user's account activity suggests they would benefit from those features. These are not generic banners; they are scored by estimated impact.

**Inline feature gating** — When a user tries to use an Action Button or Segment field, the form shows an upgrade prompt with plan details. This is a warm handoff moment — if the user asks CS about it, they are already engaged.

**JTBD recommendation cards** — The recommendations engine scores next-best-actions and can surface upsell cards as part of the recommendation set. These are weighted by potential impact, so high-fit upsells appear above low-fit ones.

When a user mentions any of these prompts on a CS call, treat it as a qualified upsell signal — not just a support question.

### Success Metrics to Track

Report on these metrics to measure extension adoption and health:

| Metric | What It Tells You |
|--------|------------------|
| Campaigns created via extension vs dashboard | Extension adoption rate; proxy for workflow shift |
| AI Write usage rate | Feature stickiness; correlates with retention |
| Time-to-first-campaign for new users | Onboarding effectiveness |
| Extension DAU / WAU | Engagement depth; are users returning daily or weekly |

A healthy account will show increasing extension usage as a share of total campaigns over the first 60 days. If that share is not growing, the user may not have discovered or adopted the extension yet — a proactive CS touchpoint can close that gap.

## For the Engineering Team

### Security Note: Gemini API Key

The Gemini API key in `modules/ai.js` is hardcoded for the internal beta. This is acceptable for internal team testing but **MUST be moved to a backend proxy before any public release**.

**For now:** Apply these Google Cloud Console restrictions:
- HTTP referrer: `chrome-extension://YOUR_EXTENSION_ID/*`
- Daily quota: 1000 requests/day

**Before public release:** Replace with `POST /sites/:id/generative-ai/campaign-copy` backend endpoint (see TODO in ai.js lines 8-15).

**Who can see the key:** Only team members who inspect the extension source. Push notification recipients CANNOT see or access the key.
